# FL4 OBD MONITOR — 設計仕様書

## 目的

Honda FL4 (Civic e:HEV) の OBD-II データをスマートフォンのブラウザでリアルタイム表示する。

---

## 方針

- サーバー不要。単一 HTML ファイルで完結。
- Web Bluetooth API 経由で ELM327 BLE ドングルに接続。
- Android Chrome（または HTTPS 環境）で動作。

---

## 技術スタック

| 項目 | 内容 |
|------|------|
| 実装 | HTML / CSS / JavaScript（バニラ） |
| 通信 | Web Bluetooth API → ELM327 BLE ドングル → OBD-II (ISO 15765-2 CAN) |
| ストレージ | なし（揮発性） |

---

## 表示項目

全PIDは TA=01 (18DAF101) から取得。

| 表示 | PID | 変換式 | 単位 |
|------|-----|--------|------|
| ENGINE RPM | 0x0C | (256A+B)/4 | rpm |
| SPEED | 0x0D | A | km/h |
| VBAT / CURRENT / BATTERY PWR | 0x9A | V=(256C+D)/64, I=signed16(256E+F)×0.1, P=V×I/1000 | V / A / kW |
| STATE OF CHARGE | 0x5B | 100A/255 | % |
| SYS V | 0x42 | (256A+B)/1000 | V |
| RUNTIME | 0x1F | 256A+B | sec |

---

## CAN 通信設定

```
TA_BATT  : 18DAF101  (常時ポーリング対象)

ELM327 初期化シーケンス:
  ATE0  — エコーオフ
  ATL0  — 改行なし
  ATS0  — スペースなし
  ATSP0 — プロトコル自動
  ATSH18DAF101   — デフォルトヘッダー (TA_BATT)
  ATFCSH18DAF101 — フローコントロールヘッダー
  ATFCSD300000   — フローコントロールデータ
  ATFCSM1        — フローコントロールモード1

起動時のみ:
  011F (TA_BATT のまま取得)
```

---

## ポーリング方式

レスポンス駆動（`>` 受信時に次PIDを即送信）+ ウォッチドッグ200ms。

```
ポーリングキュー: 010C → 019A → 010D → 015B → 0142 → (繰り返し)
```

---

## PID 0x9A (マルチフレーム) パース仕様

ELM327 独自フォーマットで返る。

```
"008 0:419A07004245 1:000D55555555"
       ├─ FF (First Frame)  : 0x41 0x9A [A][B][C][D]
       └─ CF (Consecutive)  : [E][F][pad...]

VBAT [V]    = (256×C + D) / 64
IBAT [A]    = signed16(256×E + F) × 0.1
PBAT [kW]   = VBAT × IBAT / 1000
```

FFとCFが `>` をまたいで別ブロックに分かれることがある。`pendingFF` 変数で前ブロックのFFを保持して次ブロックのCFと結合する。

## BLE サービス UUID

| Type | Service UUID |
|------|-------------|
| A (一般的な中華ドングル) | `0000fff0-0000-1000-8000-00805f9b34fb` |
| B (iOS向けドングル) | `e7810a71-73ae-499d-8c15-faa9aef0c3f2` |

両方を順に試し、Notify/Write キャラクタリスティックを自動検出する。

## 機能

| 機能 | 説明 |
|------|------|
| BLE接続 | ELM327 BLEドングルを名前/サービスUUIDでスキャン・接続 |
| リアルタイム表示 | 200ms周期で描画更新 |
| ログ記録 | REC開始〜停止でタイムスタンプ付きテキストをダウンロード |
| デモモード | 未接続時に模擬データでメーターを動作確認 |
| 横向きモード | メーターカードを90°回転（端末向き変更にも自動追従） |
| スリープ抑制 | Screen Wake Lock API でディスプレイをオンに保持 |

## 制約・既知仕様

- **Android Chrome + HTTPS（またはlocalhost）必須**。iOS Safariは Web Bluetooth 非対応。
- PID 0x9A は TA=01 専用。他 ECU では応答なし。
- TA=10 (18DAF110) 固有の PID（燃料残量 0x2F、触媒温度 0x3C 等）は現状未対応。取得にはヘッダー切替が必要。
- RUNTIME (0x1F) はエンジン始動後に TA=01 で1回だけ取得。以降は JS タイマーでカウントアップ。TA 切替不要。
